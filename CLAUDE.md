# 投资分析项目 — Claude Context

## 项目概览

基于价值投资（巴菲特/段永平思想）+ 题材投资的投资分析系统。

## 核心投资原则

- **买股票=买公司**：分析标的第一性原理是"这家公司是否值得买下全部或部分"
- **能力圈**：只在理解透了的领域下注
- **安全边际**：好公司 + 好价格 = 好投资
- **市场先生**：市场是服务于你的仆人，不是指导你的导师
- **长期主义**：以年为单位持有，不试图预测短期波动

## 投资流程

### 价值投资分析流程
1. 理解生意（商业模式、护城河）
2. 评估管理层（诚信、能力、股东导向）
3. 分析财务（ROE、现金流、负债）
4. 估算内在价值（DCF、PE、可比估值）
5. 确定安全边际
6. 制定买入/卖出计划

### 题材投资分析流程
1. 确认题材级别（政策/产业/事件）
2. 判断持续性（短期/中期/长期）
3. 锁定核心受益标的
4. 制定进出策略
5. 设置止损止盈

## 文件命名规范

- 标的分析：`[代码]-[名称]-[日期].html`，如 `600519-贵州茅台-2026-07-15.html`
- 题材分析：`[题材名]-[日期].md`，如 `低空经济-2026-07-15.md`
- 操作日志：`YYYY-MM.md`，按月归档

## 模板位置

- 标的分析模板：`01-价值投资/标的分析/模板.md`
- 风控规则：`03-风险与操作/风控规则.md`
- 题材分析模板：`02-题材投资/题材追踪/模板.md`

## ⚠️ 分析报告生成规范（重要，勿违反）

### HTML 报告要求
1. **浅色背景**（白底，不要用深色主题——用户明确要求过）
2. **折叠展开**：每个章节标题可点击折叠（JS toggle），二级细节用 `<details>/<summary>`
3. **内容深度**：不能太简略，每个维度要充分展开。必须包含：
   - 竞争格局分析（至少一张对比表）
   - 段永平"三问"测试
   - 未来12个月关键催化时间线
   - 多方法估值（PE/PS/DCF毛估估）
   - 具体的操作区间（买入/观察/高估）
   - 仓位和建仓节奏
4. **文件命名**：`[代码]-[名称]-[日期].html`，存放到 `01-价值投资/标的分析/`

### 报告提交
1. 生成 HTML 后更新 `标的分析/README.md` 索引表
2. git commit + push 到 GitHub

### 发布到网站（stock-theme-assistant）
- 运行：`python scripts/publish_reports.py`
- 然后去 `D:\ai\stock-theme-assistant` 项目 git commit + push
- 脚本自动处理：复制文件 + 更新列表页
- ⚠️ **默认操作**：分析报告 git commit 后，自动执行发布流程（publish → stock-theme-assistant commit+push），不需要额外询问

## Windows 注意事项

- **GBK 编码问题**：PowerShell 输出 emoji 会报 `UnicodeEncodeError`。脚本中 print 要避免 emoji，或清除标题中的 emoji 字符
- 发布脚本 `publish_reports.py` 已处理好此问题
